---
title: 'Register'
api: 'POST /api/auth/register'
description: 'Register a new user account'
---

Creates a new user account with email, password, and profile information. Returns the user object along with access and refresh tokens for immediate authentication.

## Request Body

<ParamField body="email" type="string" required>
  User's email address. Must be a valid email format with maximum 320 characters.
</ParamField>

<ParamField body="fullName" type="string" required>
  User's full name. Minimum 2 characters, maximum 255 characters.
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Minimum 8 characters, maximum 100 characters.
</ParamField>

<ParamField body="phone" type="string">
  User's phone number. Maximum 20 characters. Optional.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful.
</ResponseField>

<ResponseField name="data" type="object">
  Contains the registration result.

  <Expandable title="data properties">
    <ResponseField name="user" type="object">
      The created user object.
      
      <Expandable title="user properties">
        <ResponseField name="id" type="string">
          Unique user identifier.
        </ResponseField>
        
        <ResponseField name="email" type="string">
          User's email address.
        </ResponseField>
        
        <ResponseField name="fullName" type="string">
          User's full name.
        </ResponseField>
        
        <ResponseField name="phone" type="string | null">
          User's phone number.
        </ResponseField>
        
        <ResponseField name="avatar" type="string | null">
          URL to user's avatar image.
        </ResponseField>
        
        <ResponseField name="emailVerified" type="boolean">
          Whether the email has been verified.
        </ResponseField>
        
        <ResponseField name="createdAt" type="string">
          ISO 8601 timestamp of account creation.
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="accessToken" type="string">
      JWT access token for API authentication.
    </ResponseField>
    
    <ResponseField name="refreshToken" type="string">
      Refresh token for obtaining new access tokens.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

<ResponseField name="409 Conflict">
  Email address is already registered.
  
  ```json
  {
    "success": false,
    "error": {
      "message": "Email already registered",
      "statusCode": 409
    }
  }
  ```
</ResponseField>

<ResponseField name="400 Bad Request">
  Invalid request body or validation error.
  
  ```json
  {
    "success": false,
    "error": {
      "message": "Validation error",
      "statusCode": 400,
      "details": []
    }
  }
  ```
</ResponseField>

## Example Request

```bash
curl -X POST https://api.example.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "fullName": "John Doe",
    "password": "securePassword123",
    "phone": "+1234567890"
  }'
```
